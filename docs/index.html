pki

Modern PKI operations, made human. Certificate inspection, key management, TLS probing, and enrollment protocols — no OpenSSL required.

Install GitHub

Install

From source (via Cargo)

cargo install --git https://github.com/rayketcham-lab/pki-client.git

Pre-built binaries

Download static Linux binaries from GitHub Releases. No runtime dependencies.

# Download and install (Linux x86_64, static musl binary)
curl -sL https://github.com/rayketcham-lab/pki-client/releases/latest/download/pki-x86_64-unknown-linux-musl.tar.gz | tar xz
sudo mv pki /usr/local/bin/

With post-quantum support

cargo install --git https://github.com/rayketcham-lab/pki-client.git --features pqc

Quick Start

# Inspect any PKI file (auto-detects certificates, keys, CSRs, CRLs)
pki show server.pem

# Deep forensic analysis with security grades
pki cert show server.pem --format forensic

# Check which certs expire within 30 days
pki cert expires *.pem --within 30d

# Probe a live TLS server
pki probe example.com:443

# Generate an EC key pair
pki key gen ec --curve P-256

# Build and verify a certificate chain
pki chain build server.pem

# Enter interactive shell
pki
Tip: Run pki with no arguments to enter the interactive shell with tab completion and history.

Output Formats

Every command supports four output modes via --format / -f:

text
-f text (default)
Human-readable with colors, OpenSSL-style layout
json
-f json
Machine-readable for scripting and automation
compact
-f compact
One line per cert — status dashboards
forensic
-f forensic
Deep-dive — hex dumps, security grades, RFC refs

Forensic Mode

The forensic format decodes every field, assesses cryptographic strength, and flags security concerns:

Certificate Summary: Type: End Entity (DV) Grade: A Purpose: Server CA Vendor: Let's Encrypt IDENTITY ────────────────────────────────────────────────────── Subject: CN=example.com Issuer: CN=R3, O=Let's Encrypt, C=US Serial: 04:a1:b2:c3:d4:e5:f6 CRYPTOGRAPHIC STRENGTH ──────────────────────────────────────────── Signature: ecdsa-with-SHA256 STRONG — current standard Key: EC 256-bit (P-256) STRONG — ~128-bit security (P-256) VALIDITY ───────────────────────────────────────────────────── Not Before: Jan 15 00:00:00 2026 UTC Not After: Apr 15 23:59:59 2026 UTC Lifetime: 24.7% used (67 of 90 days remaining) SECURITY ASSESSMENT ────────────────────────────────────────────── PASS Strong signature algorithm PASS Key meets minimum requirements PASS SAN extension present PASS Revocation endpoints configured

Highlights

🔒

Pure Rust

No OpenSSL, no C dependencies. Built on rustls, ring, and aws-lc-rs. Single static binary.

🔍

Forensic Analysis

Decode every certificate field. Hex dumps, security grades, RFC references, and CA vendor detection.

🌐

TLS Probing

Inspect live servers. Protocol versions, cipher suites, chain validation, and security linting.

📜

Enrollment Protocols

ACME (Let's Encrypt), EST (RFC 7030), and SCEP (RFC 8894) built in. No external tools needed.

🛡️

Compliance

Validate against FIPS 140-3, NIST guidelines, and Federal Bridge policies in a single command.

⚛️

Post-Quantum Ready

ML-DSA (FIPS 204) and SLH-DSA (FIPS 205) support behind the pqc feature flag.

Post-Quantum Cryptography

Build with the pqc feature flag to enable NIST post-quantum algorithms:

cargo install --git https://github.com/rayketcham-lab/pki-client.git --features pqc

Supported algorithms:

AlgorithmStandardType
ML-DSA-44FIPS 204Lattice-based signature
ML-DSA-65FIPS 204Lattice-based signature
ML-DSA-87FIPS 204Lattice-based signature
SLH-DSAFIPS 205Stateless hash-based signature

FIPS Mode

Pass --fips to restrict all operations to FIPS 140-3 approved algorithms. Non-compliant certificates and keys are flagged.

pki --fips cert show server.pem
pki --fips probe example.com:443
pki --fips compliance check /path/to/certs/

Shell Completions

# Bash
pki completions bash > /etc/bash_completion.d/pki

# Zsh
pki completions zsh > ~/.zfunc/_pki
echo 'fpath=(~/.zfunc $fpath)' >> ~/.zshrc

# Fish
pki completions fish > ~/.config/fish/completions/pki.fish

Building from Source

git clone https://github.com/rayketcham-lab/pki-client.git
cd pki-client
cargo build --release
# Binary at target/release/pki

Static binary (no runtime dependencies)

rustup target add x86_64-unknown-linux-musl
cargo build --release --target x86_64-unknown-linux-musl
# Fully static, deploy anywhere

Architecture

Four crates, clean separation of concerns:

CrateRole
pki-clientBinary — CLI, interactive shell, 17 subcommands
pki-client-outputLibrary — certificate formatting, OID registry
pki-probeLibrary — TLS inspection and security linting
pki-hierarchyLibrary — declarative PKI hierarchy builder
